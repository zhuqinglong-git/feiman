# 新建费曼练习原型设计说明

## 设计原则

- 贴合 iTeach 工作台：左侧导航、右侧白色圆角工作区、顶部工具区和新建练习下拉参考现有产品。
- 遵循现有新建流程：先在练习池选择练习类型，再进入命名弹窗，最后进入全屏创建弹窗。
- 信息密度适中：创建弹窗中保留清晰字段和状态，不做营销式说明。
- 按业务分区：费曼内容、费曼规则、预览确认分区呈现。
- 状态可见：草稿、规则待确认、规则已确认、提交质检等状态直接展示。
- 低龄使用侧可感知：右侧保留小学堂小程序预览，帮助教研检查学生侧展示效果。
- 报告分层：老师端先呈现班级诊断和待跟进对象，再下钻到单个学生证据；家长端用小程序预览确认对外表达。
- 工作台密度：控件高度约 32px，表格行高约 56px，面板边框轻、阴影弱。

## 页面结构

- 左侧：iTeach 工作台导航，按截图保留云盘、课后练习、能力测评、素养打卡营、题库组卷等分组，底部固定回收站。
- 主页面：集团练习列表，包含返回、面包屑、移动/复制/删除、搜索、新建文件夹、新建练习、资源表格和分页。
- 命名弹窗：小型居中弹窗，标题为“新建练习”，输入名称后进入创建弹窗。
- 全屏创建弹窗：顶部标题栏、左侧步骤、中心配置区、右侧小学堂预览、底部固定操作栏。
- 报告页：报告概览、班级诊断、单个学生报告、审核记录、家长端预览和发布检查。

## 控件规范

- 新建练习下拉：白色浮层、轻阴影、行高约 36px，新增“费曼练习”作为练习类型。
- 命名弹窗：宽度约 420px，遮罩层、右上角关闭、底部取消/确定。
- 分段控件：用于“题目 / 图片 / 视频”内容类型切换。
- 标签页：用于“对话生成规则 / 引用题库解析”流程切换。
- 按钮：主要动作使用 iTeach 蓝色，次要动作使用白底描边。
- 状态标签：草稿、待确认、已确认等使用小型胶囊标签。
- 表格：集团练习列表使用文件/练习图标，展示名称、类型、绑定次数、复制次数、创建人、最近修改和操作列。
- 主色：使用 iTeach 蓝 `#0878F8`，选中态使用浅蓝底。
- 页面背景：使用浅灰 `#F3F4F6`，主工作区使用白色圆角面板。
- 左侧菜单：图标使用统一线性风格，文字单行显示，选中项为浅蓝底 + 蓝色图标文字。

- AI 规则多轮反馈调整的消息流设计。
- 题库检索结果为空、无答案解析、解析质量不足等异常状态。
- 模板质检页面中的模拟练习和评分合理性评价。
